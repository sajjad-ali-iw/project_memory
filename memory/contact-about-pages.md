## GOTCHA: pages get FLATTENED (pattern baked into DB)
The Contact page (id 22) content had been **flattened** — the `<!-- wp:pattern indexworld/contact /-->` reference was replaced in the DB with the pattern's static HTML output (old image URLs hard-coded, and `<strong>` degraded to `<p>`). So pattern-file edits stopped showing. SAME gotcha as [[frontpage-db-template-override]] but for a normal page (happens if the page is opened/saved in the block editor). **Fix: reset the page content back to the bare pattern ref** via `wp_update_page(id, content='<!-- wp:pattern {"slug":"indexworld/contact"} /-->')` — restores live-pattern rendering. If a page's edits ever stop reflecting, `curl` the page and compare image/text refs to the pattern file; if they diverge, the page is flattened — reset it. (Diagnose with `wp_get_page` — flattened = full HTML; healthy = just the wp:pattern comment.) **NOTE the 5/6 mapping is counter-intuitive (5=UK, 6=USA)** — verify the picture matches the country after any re-pull. `$cimg` helper adds `?v=filemtime`; when swapping files, `touch` them (same-second mtimes produce identical `?v` and the browser keeps the cached image).
